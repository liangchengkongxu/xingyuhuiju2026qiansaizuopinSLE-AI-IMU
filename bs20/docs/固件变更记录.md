# BS20 星闪固件变更记录（给主控同事）

## 2026-07 当前量产行为

### 拍柄固件做了什么

1. **星闪非连接扫播**，设备名 `paibing_imu`
2. **载荷格式**：ASCII 行（与 BLE Notify 一致），**不是** `EB 1A` 二进制
3. **ADV 和 Scan Response 都带** 同一份 ASCII（0xFF 厂商域）
4. **100ms 更新一帧** IMU；每帧通过 **异步 stop → set_data → start** 刷新空口
5. 广播间隔 **5ms**（125us×0x28）
6. 功率：`sle_customize_max_pwr(8,2)`，约 6～8 dBm

### 主控必须配合修改

| # | 事项 | 原因 |
|---|------|------|
| 1 | 改解析为 **ASCII 行** | 二进制解析会导致数值乱 |
| 2 | **ADV 和 Scan Response 都尝试** 取 0xFF | 双份提高收包率 |
| 3 | 按 **`@uptime_ms` 去重** | 5ms 广播导致同一帧重复多次 |
| 4 | 预期 **10Hz 有效样本** | 不是每条扫描日志都是新数据 |
| 5 | 不要用 `fota.fwpkg` 烧录拍柄 | 仅 `*_all.fwpkg` |

### 曾试验但已回退的方案

| 方案 | 问题 |
|------|------|
| 22B 二进制 `EB 1A 02` | 主控 ASCII 解析错位；数值异常 |
| 热更新 `set_announce_data` | 空口不刷新，uptime 卡死 |
| 180ms 限流 restart | 数据刷新变慢，同一帧重复过久 |
| 仅 ADV 无 ScanRsp 数据 | 远距离「有名无数据」 |
