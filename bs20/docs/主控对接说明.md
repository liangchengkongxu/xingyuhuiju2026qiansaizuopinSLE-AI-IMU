# 主控对接说明

本文描述主控面板如何接收并解析拍柄 IMU 数据。

---

## 1. 设备识别

| 字段 | 值 |
|------|-----|
| 设备名 | `paibing_imu` |
| MAC 前缀 | `cc:ad:c9:00:22:xx`（末字节 01～06 区分多把拍柄） |
| 上报周期 | 100 ms |

---

## 2. 星闪版（推荐多设备场景）

### 2.1 工作模式

- 模式：`SLE_ANNOUNCE_MODE_NONCONN_SCANABLE`（不可连接、可扫描）
- **不需要建立连接**
- 支持 40+ 把拍柄同时扫播

### 2.2 广播包结构

| 包类型 | 内容 |
|--------|------|
| **ADV（必须解析）** | 厂商特定数据 **AD Type = 0xFF**，载荷为 ASCII 九轴行 |
| Scan Response | TX Power + 完整设备名 `paibing_imu`（**无 IMU 数据**） |

> **常见误区**：扫到设备名只说明 Scan Response 收到了；IMU 在 ADV 里。距离远时短包能收到、长 ADV 可能丢包，表现为「有名无数据」。

### 2.3 主控解析步骤

1. 开启星闪扫描（Seek）
2. 过滤 `complete_local_name == "paibing_imu"` 或 MAC 前缀
3. 读取该次广播的 **ADV 原始数据**（不是 Scan Response）
4. 查找 AD 结构：`length | type=0xFF | company_id(2B) | payload`
5. 将 payload 按 UTF-8/ASCII 字符串解析，找以 `@` 开头、`\n` 结尾的行

### 2.4 与旧版二进制格式区别

旧版曾使用 22 字节二进制帧（`EB 1A 02 01 ...`）。**当前固件已改为 ASCII 行**，主控需更新解析逻辑，不可再按 `EB 1A` 头解析。

---

## 3. 蓝牙版

### 3.1 工作模式

- 可连接不可扫描广播（连接后停止广播）
- 连接 GATT 服务后通过 **Notify** 接收数据

### 3.2 主控解析步骤

1. 扫描 BLE 设备 `paibing_imu`
2. 建立 GATT 连接
3. 订阅 Notify 特征（基于 SDK `ble_uart_server` 服务）
4. 按行解析 Notify  payload（格式与星闪相同）

---

## 4. 数据行格式

```
@<uptime_ms>,A<ax>,+<ay>,+<az>,G<gx>,<gy>,R<roll>,P<pitch>,M<mag>\n
```

### 示例

```
@185230,A-13,+1,+98,G-2,-1,R+123,P-456,M102\n
```

### 字段定义

| 字段 | 类型 | 说明 |
|------|------|------|
| uptime_ms | uint | `@` 后至第一个 `,`，开机相对毫秒 |
| A 段 | 3×int | 加速度 mg÷10，带符号，如 `A-13,+1,+98` |
| G 段 | 2×int | 陀螺 dps，X/Y 两轴，如 `G-2,-1` |
| R | int | Roll 角度×10，如 `R+123` → 12.3° |
| P | int | Pitch 角度×10 |
| M | uint | 力度（三轴加速度合成）×100 |

### 参考解析伪代码（C）

```c
int parse_line(const char *line, sensor_t *out)
{
    unsigned long ms;
    long ax, ay, az, gx, gy;
    int roll, pitch;
    unsigned long mag;

    if (line[0] != '@') return -1;
    if (sscanf(line, "@%lu,A%+ld,%+ld,%+ld,G%+ld,%+ld,R%+d,P%+d,M%lu",
               &ms, &ax, &ay, &az, &gx, &gy, &roll, &pitch, &mag) != 9)
        return -1;

    out->uptime_ms = (uint32_t)ms;
    out->ax_mg = ax * 10;
    out->ay_mg = ay * 10;
    out->az_mg = az * 10;
    out->gx_mdps = gx * 1000;
    out->gy_mdps = gy * 1000;
    out->roll_deg = roll / 10.0f;
    out->pitch_deg = pitch / 10.0f;
    out->magnitude = mag / 100.0f;
    return 0;
}
```

## 5. RSSI 说明

扫描界面显示的「信号强度」一般为 **RSSI（接收信号强度）**，单位 **dBm**，正常为**负数**：

| RSSI | 含义 |
|------|------|
| -40 ~ -50 | 较强，近距离 |
| -70 ~ -85 | 偏弱，2 m 外常见 |
| 口语「80 多」 | 通常指 **-80 dBm**，表示弱信号，不是发射 80 dBm |

拍柄实际发射功率约 **6～8 dBm**（BS20 芯片硬件上限）。Scan Response 里的 TX Power AD（+8）是设备声明值，不是主控测到的 RSSI。

## 6. 多设备 MAC 分配

批量部署时，每把拍柄烧录不同末字节 MAC：

| 末字节 | MAC | 烧录包 |
|--------|-----|--------|
| 01 | cc:ad:c9:00:22:01 | paibing_sle_mac01_all.fwpkg |
| 02 | cc:ad:c9:00:22:02 | paibing_sle_mac02_all.fwpkg |
| … | … | … |
| 06 | cc:ad:c9:00:22:06 | paibing_sle_mac06_all.fwpkg |

主控按 MAC 末字节或设备名+MAC 区分各拍柄。
