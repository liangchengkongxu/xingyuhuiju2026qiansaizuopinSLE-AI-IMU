# 主控（SS928）对接说明

> **固件版本**：2026-07 星闪扫播版（ASCII 行 + 异步重启 + ADV/ScanRsp 双份）  
> **拍柄烧录包**：`paibing_sle_mac01_all.fwpkg` … `mac06_all.fwpkg`

---

## 一、主控必须做的事（Checklist）

- [ ] **解析 ASCII 行**，不要按 `EB 1A` 二进制解析（已废弃）
- [ ] 从 **ADV 或 Scan Response** 的 `0xFF` 厂商域取载荷（两路内容相同）
- [ ] 按 **`@` 后 uptime_ms 去重**：同一 MAC + 同一 `@时间戳` 只处理一次
- [ ] 过滤设备名 `paibing_imu` 或 MAC 前缀 `cc:ad:c9:00:22:xx`
- [ ] 预期 **约 100ms 一条新数据**（10Hz）；日志里同一行重复多次是正常的
- [ ] RSSI 为 **负数 dBm**（如 -80），不是发射功率

---

## 二、当前固件行为（拍柄侧）

| 项目 | 值 |
|------|-----|
| 模式 | 星闪非连接可扫描广播 |
| 设备名 | `paibing_imu` |
| MAC 前缀 | `cc:ad:c9:00:22:xx` |
| 采样/换帧 | **100ms**（10Hz） |
| 广播间隔 | **5ms**（空口发包密，主控会扫到重复帧） |
| 发射功率 | ~6～8 dBm（`sle_customize_max_pwr(8,2)`） |

### 广播包结构

| 包类型 | 内容 |
|--------|------|
| **ADV** | Discovery + Access + **0xFF 厂商域（ASCII 九轴行）** |
| **Scan Response** | TX Power + 设备名 + **0xFF 厂商域（同一份 ASCII 行）** |

拍柄每 100ms 更新 IMU 后 **异步停播→换数据→再开播**（热更新无效，必须 restart 才能刷新空口）。

---

## 三、数据格式（与 BLE Notify 完全相同）

每帧一行 ASCII，以 `\n` 结尾：

```
@10222,A-46,+32,+77,G-119,-248,R+169,P-244,M96\n
```

### 字段说明

| 字段 | 示例 | 含义 | 换算 |
|------|------|------|------|
| `@10222` | uptime | 开机相对时间 | 毫秒 |
| `A-46,+32,+77` | 加速度 X/Y/Z | 数值×10 = **mg** | -460, +320, +770 mg |
| `G-119,-248` | 陀螺 X/Y | **dps**（无 Z 轴） | -119, -248 °/s |
| `R+169` | Roll | 数值÷10 = **度** | +16.9° |
| `P-244` | Pitch | 数值÷10 = **度** | -24.4° |
| `M96` | 力度 | 数值÷100 = **g** | 0.96 g |

### 解析步骤

1. 星闪扫描（Seek），拿到 ADV 和/或 Scan Response 原始字节
2. 遍历 AD 结构：`length | type | value...`
3. 找 **`type == 0xFF`**（Manufacturer Specific Data）
4. 将 payload 当作 **UTF-8/ASCII 字符串**，找以 `@` 开头、`\n` 结尾的行
5. **`sscanf` 或等价解析**（见下方代码）

---

## 四、主控必改：去重逻辑

因广播间隔 5ms，**同一 `@时间戳` 会被扫到 2～20 次**，例如：

```
@11099,A-75,...   ← 同一条
@11099,A-75,...   ← 重复，应丢弃
@11099,A-75,...   ← 重复，应丢弃
@11398,A-47,...   ← 新时间戳，处理
```

**推荐做法**（每个 MAC 维护 last_uptime）：

```c
static uint32_t g_last_uptime[40];  /* 按 MAC 索引 */

bool paibing_should_accept(const uint8_t mac[6], uint32_t uptime_ms)
{
    int idx = mac[5] - 1;  /* 或按 MAC 查表 */
    if (idx < 0 || idx >= 40) return true;
    if (uptime_ms == g_last_uptime[idx]) {
        return false;  /* 重复帧，丢弃 */
    }
    g_last_uptime[idx] = uptime_ms;
    return true;
}
```

只把 **uptime 变化** 的帧送给业务层，日志就不会刷屏，且保持 10Hz 有效数据率。

## 五、参考解析代码（C）

```c
typedef struct {
    uint32_t uptime_ms;
    int32_t ax_mg, ay_mg, az_mg;
    int32_t gx_dps, gy_dps;
    float roll_deg, pitch_deg;
    float magnitude_g;
} paibing_imu_sample_t;

/* 从 0xFF 厂商 payload 提取 ASCII 行并解析 */
int paibing_parse_ascii_line(const char *line, paibing_imu_sample_t *out)
{
    unsigned long ms, mag;
    long ax, ay, az, gx, gy;
    int roll, pitch;

    if (line[0] != '@') return -1;
    if (sscanf(line, "@%lu,A%+ld,%+ld,%+ld,G%+ld,%+ld,R%+d,P%+d,M%lu",
               &ms, &ax, &ay, &az, &gx, &gy, &roll, &pitch, &mag) != 9) {
        return -1;
    }
    out->uptime_ms = (uint32_t)ms;
    out->ax_mg = ax * 10;
    out->ay_mg = ay * 10;
    out->az_mg = az * 10;
    out->gx_dps = gx;
    out->gy_dps = gy;
    out->roll_deg = roll / 10.0f;
    out->pitch_deg = pitch / 10.0f;
    out->magnitude_g = mag / 100.0f;
    return 0;
}

/* 从广播原始 AD 字节中提取 0xFF 载荷 */
const uint8_t *paibing_find_mfg_payload(const uint8_t *ad, uint16_t ad_len,
    uint16_t *out_len)
{
    uint16_t idx = 0;
    while (idx < ad_len) {
        uint8_t field_len = ad[idx];
        if (field_len == 0 || idx + 1 + field_len > ad_len) break;
        if (ad[idx + 1] == 0xFF) {
            *out_len = (uint16_t)(field_len - 1);
            return &ad[idx + 2];
        }
        idx = (uint16_t)(idx + 1 + field_len);
    }
    return NULL;
}
```

### 完整处理流程伪代码

```c
void on_sle_scan_result(const sle_seek_result_t *r)
{
    paibing_imu_sample_t sample;
    const uint8_t *payload;
    uint16_t plen;
    char line[96];

    /* 1. 过滤 MAC / 名字 */
    if (!mac_prefix_match(r->addr, 0xCC, 0xAD, 0xC9, 0x00, 0x22)) return;

    /* 2. 先试 ADV，再试 Scan Response */
    payload = paibing_find_mfg_payload(r->adv_data, r->adv_len, &plen);
    if (payload == NULL) {
        payload = paibing_find_mfg_payload(r->scan_rsp, r->scan_rsp_len, &plen);
    }
    if (payload == NULL || plen < 10) return;

    /* 3. 拷贝为字符串，找 @ ... \n */
    if (plen >= sizeof(line)) plen = sizeof(line) - 1;
    memcpy(line, payload, plen);
    line[plen] = '\0';

    /* 4. 解析 + 去重 */
    if (paibing_parse_ascii_line(line, &sample) != 0) return;
    if (!paibing_should_accept(r->addr, sample.uptime_ms)) return;

    /* 5. 送给业务 */
    paibing_on_imu_sample(r->addr, &sample);
}
```

## 六、与旧版差异（请勿再用旧逻辑）

| 项目 | 旧版（已废弃） | **当前固件** |
|------|----------------|-------------|
| 载荷格式 | 22B 二进制 `EB 1A 02` | **ASCII 行** `@ms,A...,G...,R...,P...,M...\n` |
| 数据位置 | 仅 ADV | **ADV + Scan Response 均有** |
| 刷新方式 | 停播/热更新多种试验 | **100ms 异步 restart** |
| 主控解析 | 按字节偏移读 int16 | **sscanf ASCII 行** |

若主控仍按 `EB 1A` 头解析，会出现 **数值错乱、G/R 异常大** 等问题。

---

## 七、蓝牙版（参考，非当前扫播路径）

蓝牙版仍为连接后 **GATT Notify**，格式与上表 ASCII 行相同，无需从广播解析。

---

## 八、RSSI 与距离

- 显示「80 多」通常是 **-80 dBm**（弱信号）
- 2m 外丢包时：优先保证去重逻辑正确，再优化扫描占空比
- 扫到名字 ≠ 收到 IMU：名字在 Scan Response，数据在 0xFF 域

---

## 九、多设备 MAC

| 末字节 | MAC |
|--------|-----|
| 01～06 | `cc:ad:c9:00:22:01` … `:06` |

按 MAC 末字节或完整 MAC 区分拍柄，**去重表按 MAC 独立维护**。
