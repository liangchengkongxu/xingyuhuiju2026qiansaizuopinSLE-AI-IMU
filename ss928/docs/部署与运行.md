# SS928 部署与运行

## 板端目录（量产后）

```
/opt/widget_ui/
├── widget_panel          # Qt 主程序
├── run.sh                # 启动脚本（含 IMU 环境变量）
├── models/badminton_npu.om
└── ws73/
    ├── sle_seek_print_all    # 星闪扫描 + IMU 解析
    ├── sle_imu_bridge.sh     # IMU 桥接守护
    ├── sle_seek_bridge.sh    # 设备扫描（绑定页）
    └── ws73_common.sh        # ko 加载、WiFi 共存
```

运行时文件：

| 路径 | 说明 |
|------|------|
| `/tmp/sle_imu_lines` | IMU `@` 行管道 |
| `/tmp/widget_imu.log` | 挥拍/CNN 调试日志 |
| `/tmp/sle_imu_bridge.pid` | 桥接进程 |

## 从开发机部署

在 SS928 工程 `version3.0/scripts/` 下：

```bash
# 星闪工具（sle_seek_print_all）
bash deploy_ws73.sh

# Qt 面板（增量）
bash deploy_panel.sh imu_swing_detector.cpp sle_imu_service.cpp

# 全量（面板 + ws73 + 脚本）
bash deploy.sh
```

环境变量（`scripts/run.sh`）：

| 变量 | 默认 | 含义 |
|------|------|------|
| `WIDGET_HIT_SOURCE` | `both` | 击球来源：imu/camera/both |
| `WIDGET_IMU_HIT_MODE` | `both` | rule/cnn/both |
| `WIDGET_IMU_RULE_SWING_ON_DYN` | `8` | 规则触发 dyn（0.08g） |
| `WIDGET_IMU_RULE_CONFIRM` | `2` | 连续确认帧数 |
| `WIDGET_IMU_RULE_COOLDOWN_MS` | `700` | 规则冷却 |
| `WIDGET_WS73_BSLE_MAX_COEX` | `1` | BSLE/WiFi 共存优先 |

## 启动顺序

`run.sh` 典型流程：

1. WiFi 看门狗（可选）
2. `sle_imu_bridge.sh start` — 后台扫描 IMU
3. `widget_panel` — Qt UI

手动重启 IMU：

```bash
/opt/widget_ui/ws73/sle_imu_bridge.sh stop
/opt/widget_ui/ws73/sle_imu_bridge.sh start
```
