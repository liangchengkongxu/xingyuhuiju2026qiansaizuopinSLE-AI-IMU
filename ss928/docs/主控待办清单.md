# SS928 主控待办清单

拍柄（BS20）固件已更新为 **22B 二进制远距离版**，主控请按此清单修改。

**详细协议见** [`../../bs20/docs/主控对接说明-远距离二进制版.md`](../../bs20/docs/主控对接说明-远距离二进制版.md)。

## 必做

1. **解析格式改为 22 字节二进制**  
   头 `EB 1A 02`，按偏移读 int16/u16，见二进制版文档中的 `paibing_parse_mfg22()`。

2. **停止 ASCII sscanf 解析星闪扫播**  
   当前星闪已改回二进制；ASCII 解析会导致 G/R 等字段错乱。

3. **从 ADV 和 Scan Response 两处取 0xFF 厂商数据**  
   内容相同，任一路成功即可。

4. **实现去重**  
   同一 MAC + 同一 **uptime（偏移 4～5）** 只上报一次业务层。  
   扫描日志可保留重复，业务只用 10Hz 新帧。

5. **验证 10Hz**  
   去重后 uptime 应约每 100ms 递增；若跳变 >200ms 为丢包，非固件重复。

## 建议

- 扫描窗口/占空比尽量大，多设备时按 MAC 分表去重
- RSSI 按负数 dBm 理解（-80 = 偏弱）
- 联调烧录包：`paibing_sle_mac01_all.fwpkg`（MAC `cc:ad:c9:00:22:01`）
- UI 若仍要 ASCII 显示，主控本地格式化（见二进制版文档第五节）

## 参考日志（正常现象）

```
uptime=11099 ax=-75 ...   ← 有效
uptime=11099 ax=-75 ...   ← 重复，去重丢弃
uptime=11398 ax=-47 ...   ← 有效（约 +300ms）
```
