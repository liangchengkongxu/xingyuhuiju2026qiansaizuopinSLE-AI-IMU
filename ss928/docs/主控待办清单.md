# SS928 主控待办清单

拍柄（BS20）固件已更新，主控请按此清单修改。**详细协议见** [`../../bs20/docs/主控对接说明.md`](../../bs20/docs/主控对接说明.md)。

## 必做

1. **解析格式改为 ASCII 行**  
   `@uptime,Aax,ay,az,Ggx,gy,Rroll,Ppitch,Mmag\n`  
   与 [`../../bs20/docs/主控对接说明.md`](../../bs20/docs/主控对接说明.md) 第五节代码一致。

2. **停止按 `EB 1A 02` 二进制解析**  
   当前星闪扫播已改回 ASCII，二进制会导致 G/R 等字段错乱。

3. **从 ADV 和 Scan Response 两处取 0xFF 厂商数据**  
   内容相同，任一路成功即可。

4. **实现去重**  
   同一 MAC + 同一 `@uptime_ms` 只上报一次业务层。  
   扫描日志可保留重复，业务只用 10Hz 新帧。

5. **验证 10Hz**  
   去重后 uptime 应约每 100ms 递增；若跳变 >200ms 为丢包，非固件重复。

## 建议

- 扫描窗口/占空比尽量大，多设备时按 MAC 分表去重
- RSSI 按负数 dBm 理解（-80 = 偏弱）
- 联调烧录包：`paibing_sle_mac01_all.fwpkg`（MAC `cc:ad:c9:00:22:01`）

## 参考日志（正常现象）

```
@11099,A-75,...   ← 有效
@11099,A-75,...   ← 重复，去重丢弃
@11398,A-47,...   ← 有效（约 +300ms）
```
