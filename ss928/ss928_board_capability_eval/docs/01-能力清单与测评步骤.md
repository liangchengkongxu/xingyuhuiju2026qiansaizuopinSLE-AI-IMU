# 01 — SS928 能力清单与测评步骤

平台：**海思 SS928V100**（A55 + 媒体子系统 + NPU）  
常见外设：MIPI 摄像头、HDMI/VO 显示、GFBG 图形层、WS73 星闪模组、有线网口。

本清单用于 **离线摸底**：每项记录「通过 / 失败 / 未测」及现象即可。

---

## A. 基础系统

| # | 测评项 | 方法 | 期望 |
|---|--------|------|------|
| A1 | 网线同网段 | PC `ping BOARD_IP` | 通 |
| A2 | SSH | `bash scripts/check_board.sh` | 可登录 |
| A3 | CPU 架构 | `uname -m` | `aarch64` |
| A4 | 磁盘空间 | `df -h /` | 有可用空间；注意 `/` 写满会导致日志失败 |
| A5 | 板端 gcc | `gcc --version` | 有则便于板上编译 |
| A6 | 一键快照 | `bash scripts/probe_board_capability.sh` | 生成报告 |

---

## B. 显示与触摸

| # | 测评项 | 方法 | 期望 |
|---|--------|------|------|
| B1 | framebuffer | `ls -l /dev/fb0`；`cat /sys/class/graphics/fb0/virtual_size` | 有 fb0 |
| B2 | 分辨率/色深 | probe 脚本输出；或 `fbset`（若有） | 常见 1920×1080 |
| B3 | GFBG 与用户态 UI | 见 docs/04；BUF_NONE + yres_virtual=yres | Qt/linuxfb 可稳定常驻 |
| B4 | 触摸设备 | `ls /dev/input/event*`；`evtest`（若有） | 有触控节点 |

## C. 媒体通路（需 SDK sample）

| # | 测评项 | 方法 | 期望 |
|---|--------|------|------|
| C1 | 原厂 VIO 预览 | SDK `smp/.../sample/vio` 编译运行 | HDMI 出图 |
| C2 | GFBG sample | `sample/gfbg` | 图形层叠加 |
| C3 | 编码 VENC | `sample` 中 venc 相关 | 可出码流 |
| C4 | VB/SYS 释放 | 异常后 reboot | `vb_set_conf` 不再失败 |

说明：媒体子系统占用后进程异常退出，常需 **reboot** 才能干净初始化。

## E. 星闪 WS73

| # | 测评项 | 方法 | 期望 |
|---|--------|------|------|
| E1 | 原厂目录 | `ls /opt/sample/ws73/*.ko` | `plat_soc.ko` / `sle_soc.ko` |
| E2 | mcu_tool 射频 | 见 docs/03 | nl on 后 ko 可加载 |
| E3 | 与 WiFi 共存 | 测星闪时按需关 WiFi | 减少冲突 |
| E4 | 吞吐 | `sle_throughput_test/deploy_board.sh` | 有稳定 Kbps 输出 |

经验参考（仅作基线，非指标承诺）：1M PHY + SSAP write 峰值约数 Mbps 量级。
