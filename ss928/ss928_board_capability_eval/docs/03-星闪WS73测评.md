# 03 — 星闪 WS73 测评

模组常见路径：`/opt/sample/ws73/`（`plat_soc.ko`、`sle_soc.ko`、`wifi_sta.sh`、`mcu_tool`）。

## 上电与加载驱动（板端）

```bash
killall sle_tp_client 2>/dev/null
/opt/sample/ws73/wifi_sta.sh 1      # 按需关 WiFi，减少共存干扰
rmmod sle_soc plat_soc 2>/dev/null
mcu_tool /dev/i2c-0 0x10 nl off
sleep 0.5
mcu_tool /dev/i2c-0 0x10 nl on
sleep 1
insmod /opt/sample/ws73/plat_soc.ko
insmod /opt/sample/ws73/sle_soc.ko
lsmod | grep -E 'plat_soc|sle_soc'
```

## PC 侧 WS73 SDK

吞吐程序链接 `libsle_host.a`，需解压原厂 SDK：

```
WS73_SDK/application/lib/rk3568/sle/libsle_host.a
WS73_SDK/include/bsle/ ...
```

查找顺序（`scripts/board_common.sh`）：

1. 环境变量 `WS73_SDK`
2. 本包旁 `../星闪WS73原厂SDK/extracted`
3. 本包内 `ws73_sdk/extracted`

仅做连通性检查时可不配 SDK；部署测速前必须配好。

## 吞吐测速

```bash
# 改对端 MAC：编辑 sle_throughput_test/sle_tp_common.h 后重新 deploy
bash scripts/check_board.sh
bash sle_throughput_test/deploy_board.sh

# 板端
/opt/sample/ws73/sle_tp_run.sh 0
/opt/sample/ws73/sle_tp_run.sh 0 -s 251 -i 1000
/opt/sample/ws73/sle_tp_run.sh 1
```

| 项 | 说明 |
|----|------|
| 角色 | 本板为 TX Client；对端需 GATT Server |
| UUID | 服务 `0x2222` / 可写特征 `0x2323` |
| payload | 默认 251，最大 251 |
| PHY | 本端可请求 4M，实际常协商为 1M（看对端） |

可选接收端：

```bash
DEPLOY_SERVER=1 bash sle_throughput_test/deploy_board.sh
/opt/sample/ws73/sle_tp_server -i 1000
```

## 常见错误

| 现象 | 建议 |
|------|------|
| `ssap err 0x0b` | 对端未起对应 GATT 服务 |
| `TX window: 0 bytes` | 未发现可写特征 |
| `disconnected reason=0x7` | 链路/发现超时 |
| 找不到 `libsle_host.a` | 设置 `WS73_SDK` |
| 装 ko 失败 | 先 `nl on`，再 `insmod` |
