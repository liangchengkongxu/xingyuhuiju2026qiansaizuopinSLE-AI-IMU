# BS20 拍柄固件 · 开发过程与代码要点

本文记录星羽汇聚 2026 嵌赛 BS20 拍柄端从 0 到联调可用的**开发历程**与**关键代码决策**，便于答辩、交接与后续维护。

---

## 1. 项目目标

| 目标 | 实现 |
|------|------|
| 读取 MPU-9250 九轴 | 软件 I2C + Mahony AHRS |
| 40+ 把拍柄同时上报 | 星闪非连接扫播（不占连接数） |
| 主控 SS928 解析 | 22 字节 `EB 1A 02` 二进制帧 |
| 备用蓝牙调试 | BLE 连接 + GATT Notify ASCII 行 |
| 批量部署 | MAC 末字节 01～08 独立烧录包 |

## 2. 架构演进

### 2.1 最终架构（当前）

```
common/paibing_imu.c     ← MPU9250 + AHRS + 100ms 主循环
        │
        │ paibing_transport_t { init, push_sensor }
        ├─► sle/paibing_app.c      → 非连接广播（主推）
        └─► ble/paibing_app.c      → 连接 Notify（调试）
```

**设计要点**：IMU 与无线发送解耦，星闪/蓝牙仅实现 `push_sensor()`，共用采样与解算。

### 2.2 曾尝试但弃用的方案

| 方案 | 问题 | 结论 |
|------|------|------|
| 星闪连接 + Notify | 连接数有限、易断链、`send busy` 丢包 | 改回非连接广播 |
| ASCII ~50B 扫播 | 空口时间长，5m 约 80%、7m+ 差 | **改回 22B 二进制** |
| 热更新 `set_announce_data` | 播着时数据不进空口，uptime 卡首帧 | **必须异步 restart** |
| 同步 stop→sleep→start | disable 回调跟不上，0x8000600a | 改 disable 回调里 commit+start |
| 180ms 限流 restart | 破坏 10Hz，同一帧重复过久 | 每 100ms 全量 restart |
| 间隔 0x28（5ms） | 可能低于协议栈最小值 | 改 **0x50（10ms）** |
| 仅调 `announce_tx_power=20` | 未 customize 时实际只有 ~6dBm | 加 `sle_customize_max_pwr(8,2)` |

## 3. 开发里程碑

### 阶段 1：IMU 与串口验证

- 完成 MPU-9250 软件 I2C（MGPIO15/16）
- 上电 1s 陀螺零偏校准（20 样本 × 50ms）
- UART0 115200 打印 acc/gyro/roll/pitch

### 阶段 2：蓝牙 Notify 通路

- 基于 SDK `ble_uart_server` 示例
- 连接后 Notify ASCII：`@ms,A...,G...,R...,P...,M...\n`
- Target：`standard-bs20-n1200-ble`

### 阶段 3：星闪非连接扫播

- `SLE_ANNOUNCE_MODE_NONCONN_SCANABLE`
- 设备名 `paibing_imu`，MAC 前缀 `cc:ad:c9:00:22:xx`
- 载荷放在 Manufacturer AD `0xFF`

### 阶段 4：联调踩坑与修复

| 现象 | 根因 | 修复 |
|------|------|------|
| BurnTool 烧录超时 | 误用 `fota.fwpkg`（OTA 格式） | 必须用 `*_all.fwpkg` |
| 扫到名无数据 | IMU 只在 ADV，名字在 Scan Response | ADV+ScanRsp 双份数据 |
| 包头对、数据全 0 / uptime 不变 | 播着时 `set_announce_data` 不生效 | 异步 stop→disable 回调→commit+start |
| `restart fail 0x8000600a` | 同步 stop 未完成就 start | disable 回调里再 start |
| 5m 约 80%、7m+ 差 | ASCII 载荷长 + 远距离 CRC 失败 | 22B 二进制 + 0x50 间隔 |
| 主控数值错乱 | 固件二进制、主控仍 ASCII 解析 | 主控改按偏移读 int16 |
| RSSI「80多」误解 | 实际是 -80 dBm 弱信号 | 文档说明 |

### 阶段 5：远距离优化（2026-07）

- 载荷：**22B 二进制** `EB 1A 02`（ADV + ScanRsp 双份）
- 刷新：每 100ms **异步 restart**（热更新无效）
- 间隔：**0x50**（10ms 合法最小）
- 功率：`sle_customize_max_pwr(8,2)` + `announce_tx_power=20`
- 批量 MAC 扩展至 **01～08**

### 阶段 6：工程化

- `tools/build_paibing_sle.sh [MAC末字节]` 批量出包
- GitHub 仓库分 `bs20/` 与 `ss928/` 目录
- SDK 集成 overlay（方案 C）
- Cursor Skill：`.cursor/skills/paibing-bs20/SKILL.md`

## 4. 核心代码要点

### 4.1 主循环 `common/paibing_imu.c`

```c
#define PAIBING_SEND_INTERVAL_MS 100   // 10Hz 上报
// 流程：读 MPU9250 → 低通 → Mahony → push_sensor()
paibing_imu_run(const paibing_transport_t *transport);
```

- 校准完成前只采样不发送（除校准完成通知）
- 必须周期性 `uapi_watchdog_kick()`

### 4.2 星闪广播 `sle/paibing_sle_server_adv.c`

**当前策略（远距离二进制版）**：

1. `sle_customize_max_pwr(8, 2)`（在 `paibing_sle_server.c` 注册前调用）
2. 广播间隔 `0x50`（10ms）
3. **22 字节二进制帧**（非 ASCII）
4. **ADV + Scan Response 均带 0xFF 厂商数据**
5. 每 100ms：`stage` → `stop` → **`disable` 回调**里 `commit + start`
6. **禁止** 播着时单独 `sle_set_announce_data`（会卡首帧）

**刷新流程**：

```
push_sensor (100ms)
  → stage_announce_buffers (RAM 组包)
  → rf_publish: sle_stop_announce
  → announce_disable_cbk: sle_set_announce_data + sle_start_announce
```

**二进制帧布局**：

```
偏移  字段
0-1   EB 1A        厂商 ID
2     02           协议：传感器数据
3     DEV          设备号 (mac_config.h)
4-5   uptime       uint16 LE, ms 低 16 位
6-7   ax           int16 LE, mg/10
8-9   ay
10-11 az
12-13 gx           int16 LE, dps
14-15 gy
16-17 roll         int16 LE, deg×10
18-19 pitch
20-21 magnitude    uint16 LE, ×100
```

### 4.3 配置宏 `sle/mac_config.h`

```c
#define PAIBING_USE_FIXED_LOCAL_MAC 1
#define PAIBING_SLE_SENSOR_BROADCAST 1   // 1=扫播 0=连接Notify
#define PAIBING_LOCAL_MAC_B5 0x06        // 末字节，批量烧录用脚本改
```

## 6. 主控侧要点

1. 过滤设备名 `paibing_imu` 或 MAC `cc:ad:c9:00:22:xx`
2. 解析 **ADV 或 Scan Response** 中 `type=0xFF` 的 **22 字节**载荷
3. 头必须是 `EB 1A 02`（**不是** ASCII `@` 行）
4. 按 **uptime（偏移 4～5）+ MAC** 去重；10ms 广播会重复扫到同一帧
5. RSSI 为负数 dBm（-80 表示偏弱，不是发射 80dBm）
6. 蓝牙版仍用连接后 ASCII Notify

详见 [`../bs20/docs/主控对接说明-远距离二进制版.md`](../bs20/docs/主控对接说明-远距离二进制版.md)。
