# BS20 拍柄固件 · 开发过程与代码要点

本文记录星羽汇聚 2026 嵌赛 BS20 拍柄端从 0 到联调可用的**开发历程**与**关键代码决策**，便于答辩、交接与后续维护。

> **当前协议（2026-07）**：星闪扫播 **ASCII 行**（与 BLE 相同），ADV + Scan Response 双份 0xFF，主控按 `@uptime_ms` 去重。  
> 曾试验的 `EB 1A 02` 二进制方案已回退，见 [固件变更记录](../bs20/docs/固件变更记录.md)。

## 1. 项目目标

| 目标 | 实现 |
|------|------|
| 读取 MPU-9250 九轴 | 软件 I2C + Mahony AHRS |
| 40+ 把拍柄同时上报 | 星闪非连接扫播（不占连接数） |
| 主控 SS928 解析 | ASCII `@` 行 + uptime 去重 |
| 备用蓝牙调试 | BLE 连接 + GATT Notify 同格式 ASCII |
| 批量部署 | MAC 末字节 01～06 独立烧录包 |

## 2. 架构演进

### 2.1 最终架构（当前）

```
common/paibing_imu.c     ← MPU9250 + AHRS + 100ms 主循环
        │
        │ paibing_transport_t { init, push_sensor }
        ├─► sle/paibing_app.c      → 非连接广播 ASCII（量产）
        └─► ble/paibing_app.c      → 连接 Notify ASCII（调试）
```

**设计要点**：IMU 与无线发送解耦，星闪/蓝牙仅实现 `push_sensor()`，共用采样与解算。

### 2.2 协议演进摘要

| 阶段 | 载荷 | 结果 |
|------|------|------|
| 早期 | ADV 内 ASCII 长行 | 2m 外丢包多 |
| 中期 | 22B `EB 1A 02` 二进制 | 主控 ASCII 解析错位，数值异常 |
| **当前** | **ASCII 行，ADV+ScanRsp 双份** | 与 BLE 统一，主控 sscanf + 去重 |

### 2.3 曾尝试但弃用/回退的方案

| 方案 | 问题 | 结论 |
|------|------|------|
| 星闪连接 + Notify | 连接数有限、易断链 | 非连接广播 |
| 热更新 `set_announce_data` 不 restart | 空口不刷新，uptime 卡死 | **100ms 异步 stop→set→start** |
| 180ms 限流 restart | 刷新慢，重复帧过久 | 恢复 100ms |
| 仅 ADV 无 ScanRsp 数据 | 远距离「有名无数据」 | Scan Response 也带 0xFF |
| 5ms 广播无去重 | 主控日志刷屏、误触发 | **主控 MAC+uptime 去重** |

## 3. 开发里程碑

### 阶段 1：IMU 与串口验证

- MPU-9250 软件 I2C（MGPIO15/16）
- 上电 1s 陀螺零偏校准（20×50ms）
- UART0 115200 打印 acc/gyro/roll/pitch

### 阶段 2：蓝牙 Notify

- 连接后 Notify：`@ms,A...,G...,R...,P...,M...\n`
- Target：`standard-bs20-n1200-ble`

### 阶段 3：星闪非连接扫播

- `SLE_ANNOUNCE_MODE_NONCONN_SCANABLE`
- 设备名 `paibing_imu`，MAC `cc:ad:c9:00:22:xx`

### 阶段 4：联调踩坑

| 现象 | 根因 | 修复 |
|------|------|------|
| BurnTool 超时 | 误烧 `fota.fwpkg` | 用 `*_all.fwpkg` |
| 扫到名无数据 | 只解析 Scan Response | 解析 ADV/ScanRsp 的 0xFF |
| G/R 乱数 | 主控按二进制解 ASCII | 改 ASCII 解析 |
| 同一行重复 N 次 | 5ms 广播 vs 100ms 换帧 | 主控 uptime 去重 |
| RSSI「80多」 | 实际是 -80 dBm | 文档说明 |

### 阶段 5：工程化

- `tools/build_paibing_sle.sh [01-06]`
- GitHub 分 `bs20/`、`ss928/`
- 主控参考代码入 `ss928/reference/`

## 4. 核心代码要点（当前）

### 4.1 主循环 `common/paibing_imu.c`

```c
#define PAIBING_SEND_INTERVAL_MS 100   // 10Hz
paibing_imu_run(const paibing_transport_t *transport);
```

### 4.2 星闪广播 `sle/paibing_sle_server_adv.c`

1. `sle_customize_max_pwr(8, 2)` → 约 6～8 dBm  
2. 广播间隔 5ms（空口重复多，靠主控去重）  
3. **ASCII 行**放入 0xFF 厂商域  
4. **ADV + Scan Response** 各一份  
5. 每 100ms **异步 stop → set_data → start**

### 4.3 ASCII 行格式

```
@10222,A-46,+32,+77,G-119,-248,R+169,P-244,M96\n
```

A×10=mg，R/P÷10=度，M÷100=g。

## 6. 主控侧要点

1. 解析 **ASCII 行**，非 `EB 1A`  
2. ADV **与** Scan Response 均尝试取 `0xFF`  
3. **MAC + uptime 去重**，有效 10Hz  
4. 详见 [主控对接说明](../bs20/docs/主控对接说明.md)、[ss928 实现说明](../ss928/docs/IMU解析与去重实现.md)
