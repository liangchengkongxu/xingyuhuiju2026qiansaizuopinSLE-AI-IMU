# 联调快速指南

拍柄（BS20）与主控（SS928 + WS73）端到端联调步骤。

---

## 1. 硬件准备

| 设备 | 要求 |
|------|------|
| BS20 拍柄 | 已烧录 `paibing_sle_mac01_all.fwpkg`（或 02～06） |
| SS928 板 | 已插 WS73 星闪模块，可 SSH（默认 `192.168.1.168`） |
| 串口（可选） | 拍柄 UART0 115200，看 `acc(mg):... roll:...` 是否正常 |

拍柄 MAC 示例：`cc:ad:c9:00:22:01`，设备名 `paibing_imu`。

---

## 2. 拍柄自检（UART）

上电后串口应约 **1s 校准**，然后每 100ms 打印类似：

```
acc(mg):-460,320,770 gyro(mdps):... roll:16.9 pitch:-24.4 mag:0.96 @10222ms
```

- **uptime 递增** → IMU 主循环正常  
- **roll/pitch 非长期 0** → AHRS 正常  
- 若 uptime 卡住 → 查广播 restart 是否失败

---

## 3. 主控启动 IMU 桥接

SSH 登录 SS928：

```bash
/opt/widget_ui/ws73/sle_imu_bridge.sh stop
/opt/widget_ui/ws73/sle_imu_bridge.sh start
tail -f /tmp/sle_imu_lines
```

**正常输出**（去重后约 10Hz）：

```
[SLE_IMU] mac=cc:ad:c9:00:22:01|@11099,A-75,+32,+77,G-119,-248,R+169,P-244,M96
[SLE_IMU] mac=cc:ad:c9:00:22:01|@11398,A-47,...
```

### 正常现象

- 原始扫描可能同一 `@11099` 出现多次 → **C 层去重后日志应约 100ms 一条新 uptime**
- RSSI 为负数，如 `-75 dBm`

### 异常现象

| 现象 | 可能原因 |
|------|----------|
| 只有设备名、无 `[SLE_IMU]` | 只收到 Scan Response 名字，未解析到 0xFF |
| `G/R` 巨大乱数 | 主控仍按 `EB 1A` 二进制解析 → 改 ASCII |
| uptime 长期不变 | 拍柄广播未更新或主控重复读缓存 |
| 完全无设备 | WS73 ko 未加载、WiFi 互斥、扫描参数错误 |

---

## 4. 启动训练 UI

```bash
cd /opt/widget_ui && ./run.sh
```

环境变量（`run.sh` 内，可调击球灵敏度）：

| 变量 | 含义 |
|------|------|
| `WIDGET_HIT_SOURCE` | `both` / `imu` / `camera` |
| `WIDGET_IMU_HIT_MODE` | `rule` / `cnn` / `both` |
| `WIDGET_IMU_RULE_SWING_ON_DYN` | 规则触发 dyn（centi-g） |

调试日志：`tail -f /tmp/widget_imu.log`

---

## 5. 部署更新（开发机）

```bash
# 仅更新星闪扫描解析
cd <SS928工程>/version3.0/scripts
bash deploy_ws73.sh

# 仅更新 Qt 面板
bash deploy_panel.sh sle_imu_service.cpp imu_swing_detector.cpp

# 重启
ssh root@192.168.1.168 "/opt/widget_ui/ws73/sle_imu_bridge.sh stop; \
  /opt/widget_ui/ws73/sle_imu_bridge.sh start; killall widget_panel; \
  cd /opt/widget_ui && ./run.sh"
```

---

## 6. 多拍柄

- MAC 末字节 01～06 各烧一柄  
- 主控按 MAC 独立去重表（见 `ss928/docs/IMU解析与去重实现.md`）  
- 对打模式 UI 支持最多 4 台绑定
