# 联调示例

## 0. 检查服务

```bash
curl http://47.107.120.9/health
# 期望: {"status":"healthy"}
```

## 1. 上报一场对打

```bash
curl -X POST http://47.107.120.9/api/v1/device/ingest/match \
  -H "Content-Type: application/json" \
  -d @match_sample.json
```

`match_sample.json` 即 `examples/match_sample.json`（复制到当前目录或改路径）。

期望响应：

```json
{"ok":true,"matchId":"..."}
```

## 2. 上报一次练习

```bash
curl -X POST http://47.107.120.9/api/v1/device/ingest/drill \
  -H "Content-Type: application/json" \
  -d @drill_sample.json
```

## 3. 验证 App 是否可见（需登录 token）

```bash
# 登录演示用户
curl -X POST http://47.107.120.9/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"phone":"13800138000","password":"123456"}'

# 用返回的 token
curl http://47.107.120.9/api/v1/matches \
  -H "Authorization: Bearer <token>"
```

最新一条比赛应包含刚才上报的 `title`。

## 4. Windows PowerShell 单行示例

```powershell
curl.exe -X POST http://47.107.120.9/api/v1/device/ingest/drill `
  -H "Content-Type: application/json" `
  -d '{\"device_id\":\"xingyu-hi3403-01\",\"user_phone\":\"13800138000\",\"action_type\":\"smash\",\"score\":93,\"ai_suggestion\":\"完美一拍\",\"ball_speed_kmh\":175,\"power_n\":80}'
```

## 5. 常见问题

| 现象 | 处理 |
|------|------|
| 422 Validation Error | 检查必填字段、JSON 格式 |
| App 看不到数据 | 确认 `user_phone` 与 App 登录账号一致 |
| 数据出现在别人账号 | `user_phone` 填错或未填，后端 fallback 到第一个用户 |
| 连接超时 | 检查 4G 网络、服务器 80 端口、防火墙 |
